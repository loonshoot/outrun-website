Feature

AI Workflow Builder

Build multi-step automation workflows that combine AI reasoning, code execution, and conditional logic. Trigger them from GitHub webhooks, inbound email, schedules, or run them manually.

View all features →

Capabilities

Mix and match nodes to automate anything

Flexible Triggers

Start workflows from GitHub webhooks, inbound email, or cron schedules. Each trigger passes its data into your workflow steps so every node can use it.

AI & Code Nodes

AI nodes call Claude or OpenAI for reasoning tasks like summarising, classifying, or drafting. Code nodes run isolated sessions that can read, write, and commit changes to repositories.

Conditional Branching

Route your workflow down different paths using conditions like equals, contains, or greater than. Branch on AI output, trigger data, or any earlier step's result.

Reliable execution you can trace

Steps run in the correct order based on their dependencies. Each node waits for the ones it depends on to finish before it starts. Every run is logged so you can see exactly what happened and when.

✓ Steps run in the right order automatically
✓ Template variables let any step reference an earlier step's output
✓ Full run history saved for every workflow execution
✓ Skipped branches are tracked so you can see why a path was not taken

How data flows between steps

Trigger Data

Access the event that started your workflow - the webhook payload, email content, or schedule metadata

AI Output

Reference AI responses in later steps using template variables like {{ai.text}} or by node name

Code Results

Code nodes return structured results that downstream steps can use for decisions or actions

Conditional Routing

When a condition is not met, that entire branch is skipped - and the skip carries through to any steps that only depend on it
